# スクリーンショットを自動撮影する

## 🎯 このセクションで学ぶこと

- manual-screenshot-capture スキルでスクリーンショットを一括撮影する
- 撮影の流れと注意点を理解する

---

## 🏃 スクリーンショットを自動撮影する

### Step 1: 準備を確認する

| 必要なもの | 確認 |
|-----------|------|
| 📸 撮影指示付きのマニュアル | セクション 7 で作成済み？ |
| CLAUDE.md にアプリ URL とログイン情報 | 記載済み？ |
| Playwright MCP の接続 | セクション 4 で設定済み？ |

> ⚠️ **CLAUDE.md のアプリ URL とログイン情報は必須です。** 撮影スキルはブラウザでアプリにログインして画面を撮影するため、これがないと動作しません。

### Step 2: スキルを呼び出す

```
/manual-screenshot-capture docs/manual.md のスクリーンショットを撮影して
```

> 💡 `docs/manual.md` の部分は、セクション 7 で生成されたマニュアルのファイルパスに合わせてください。

スキルが起動すると、以下の流れで進みます。

1. マニュアル内の `📸 撮影指示` をすべて読み取る
2. 撮影リストを提示し、確認を待つ
3. アプリにログイン
4. マニュアルの順番に沿って 1 枚ずつ撮影・保存
5. 撮影結果をステータス付きで報告

### Step 3: 撮影結果を確認する

撮影が完了すると、以下のような報告が表示されます。

| ファイル名 | 内容 | ステータス |
|----------|------|----------|
| `3-1-1-user-list.png` | ユーザー一覧画面 | ✅ 撮影済 |
| `3-2-1-user-detail.png` | ユーザー詳細画面 | ✅ 撮影済 |
| `4-1-1-settings.png` | 設定画面 | ⚠️ 要差替 |

### 異常時は自動で止まる

撮影中に以下のような状況が起きたら、スキルは**自動で作業を止めてあなたに報告**します。

- ボタンが無効化されてクリックできない
- エラーメッセージが表示されている
- データが空で指示通りの画面を再現できない
- 期待する画面に遷移しない

報告を受けたら、Claude Code に対処方法を指示してください（例: 「テストデータを先に作成して」「そのスクショはスキップして」）。

### マニュアルも自動で直してくれる

撮影中にマニュアルの記述と実際の画面が違う場合、スキルが**マニュアルの記述を自動で修正**します。

例:
- マニュアル「画像をクリック」→ 実際は「画像を変更」ボタン → 自動修正
- 撮影指示の要素が画面上にない → 手順を修正

### Step 4: 完成したマニュアルを確認する

撮影後、`assets/` フォルダに画像が保存されています。

```
assets/
├── admin/images/
│   ├── 3-1-1-user-list.png
│   ├── 3-2-1-user-detail.png
│   └── ...
└── user/images/
    └── ...
```

テキストエディタや Markdown プレビューでマニュアルを開いて、画像が正しく表示されていれば完成です！

---

## ✨ まとめ

- `/manual-screenshot-capture` で 📸 撮影指示から自動でスクリーンショットを撮影
- 異常な画面を検知したらスキルが自動停止して報告してくれる
- マニュアルの記述と実際の画面が違えば、スキルが自動で修正する
- 全セクションの撮影が完了すれば、画像付きの操作マニュアルが完成！
