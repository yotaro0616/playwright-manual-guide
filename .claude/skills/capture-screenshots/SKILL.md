# Capture Screenshots

Markdown マニュアル内の `📸 撮影指示` コメントを読み取り、Playwright でブラウザ操作を行いスクリーンショットをファイルに保存するスキル。

## 前提

- CLAUDE.md にアプリ URL、アカウント情報が記載されていること
- `📸 撮影指示` コメント付きのマニュアルファイルが存在すること（`/create-manual` で生成）
- 上記がない場合は `/setup` → `/explore` → `/create-manual` の実行を案内して中断する

## プロセス

### ステップ 1: 準備

1. ユーザーが指定したマニュアルファイルを読み、全ての `📸 撮影指示` を抽出する
2. 撮影リスト（ファイル名 + 撮影内容）を一覧化し、ユーザーに確認を取る
3. **`browser_resize` で 1920x1080 に設定する**（必須。Playwright のデフォルトビューポートはこれより小さく、レスポンシブデザインでサイドバー等が非表示になる場合がある）

### ステップ 2: ログイン

1. CLAUDE.md からアプリ URL とアカウント情報を読み取る
2. ログインページにアクセスしてログインする
3. ログイン後のデフォルトページが表示されることを確認する

### ステップ 3: 順次撮影

マニュアルの順番に沿って撮影する。理由:
- 後のセクションの画面は、前のセクションの操作結果に依存する場合がある
- ログイン→ダッシュボード→各機能ページの順が自然な遷移になる

各スクリーンショットについて:

1. **ページ遷移** — 撮影指示に記載された画面に移動する
2. **状態再現** — 必要な操作を行う（フォーム入力、タブ切替、スクロールなど）
3. **待機** — ページの読み込み完了を待つ（2〜3 秒）
4. **撮影範囲を判断** — 下記の基準に従う
5. **撮影** — Playwright の `screenshot()` でファイルに保存する
6. **検証** — 撮影結果を確認し、意図した内容が写っているか確認する

### ステップ 4: 結果報告

撮影完了後、以下の形式でユーザーに報告する:

| ファイル名 | 内容 | ステータス |
|----------|------|----------|
| `3-1-1-dashboard.png` | ダッシュボード全体 | ✅ 撮影済 |
| `3-2-1-basic-info.png` | 基本情報セクション | ✅ 撮影済 |
| `4-5-2-form-input.png` | フォーム入力画面 | ⚠️ 要差替 |

## 撮影範囲の判断基準

**ビューポート全体で撮影する場合:**
- ページ全体のレイアウトを見せたいとき
- モーダルやダイアログが画面全体を覆っているとき
- 複数の要素の位置関係を示したいとき

**要素単位で切り抜く場合:**
- ボタン単体を示したいとき
- 特定のセクションやカード 1 つだけを見せたいとき
- ページ全体を見せると対象が小さすぎて伝わらないとき

### 要素切り抜き時の注意

- 切り抜きが狭すぎる場合: 子要素ではなく親要素の ref を指定する
- 切り抜きが広すぎる場合: コンテナ全体ではなく特定セクションの ref を指定する
- 撮影後は必ず画像を確認し、問題があれば ref を変えて撮り直す

### トースト通知の対処

操作後にトースト通知（成功メッセージ等）が画面に表示されることがある。映り込んで画像が汚れる場合は、JavaScript で非表示にしてから撮影する。

```javascript
await page.evaluate(() => {
  document.querySelectorAll('[role="status"], [class*="toast"], [class*="notification"]')
    .forEach(el => el.style.display = 'none');
});
```

## マニュアル内容の検証と修正

撮影中にマニュアルの記述と実際の UI が異なることに気づいた場合は、**マニュアルの記述を修正する**。撮影指示に従って機械的に撮るのではなく、実際の画面と照合しながら進める。

例:
- マニュアル「画像をクリック」→ 実際は「画像を変更」ボタン → マニュアルを修正
- 撮影指示に書かれた要素が画面上にない → 手順を修正

## Playwright で技術的に撮影不可能なケース

以下のケースに遭遇した場合は、**自己判断でプレースホルダー画像を作らず、ユーザーに報告し代替案を提案する**。

- **OS ネイティブダイアログ**: ファイル選択ダイアログ等
- **ドラッグ＆ドロップ中の状態**: 再現が困難
- **ホバー＋クリックの複合状態**: 特定の複合操作状態の再現が困難な場合

## 異常な画面状態の検知（最重要ルール）

撮影中に以下のような正常系でない画面状態を検知した場合は、**自己判断で進めず、必ずユーザーに確認を取る**:

- ボタンが `disabled` で操作できない
- 警告・エラーメッセージが表示されている
- データが空で撮影指示の内容を再現できない
- 期待する画面遷移が起きない

**理由**: プレースホルダー画像で進めると後戻りが発生し、大幅な時間のロスになる。

## 撮影のコツ

- **読み込み待ち**: SPA では遷移後にコンテンツが遅延ロードされることがある。最低 2 秒待ってから撮影する
- **スクロール位置**: `scrollIntoView({ behavior: 'instant', block: 'center' })` を使って対象要素までスクロールするのが最も確実。`container.scrollTop` はスクロールコンテナの特定が必要で失敗しやすい
- **ログイン状態の維持**: Playwright セッション内では Cookie が維持される。長時間の撮影ではセッションタイムアウトに注意
- **ロール切替**: 複数ロールのスクリーンショットが必要な場合は、ロールごとにまとめて撮影する
