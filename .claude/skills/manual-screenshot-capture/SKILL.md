# Manual Screenshot Capture

Markdownマニュアル内の `📸 撮影指示` コメントを読み取り、Playwrightでブラウザ操作を行いスクリーンショットをファイルに保存するスキル。

## このスキルが必要な理由

操作マニュアルには大量のスクリーンショットが必要だが、手動撮影は時間がかかり、ウィンドウサイズの不統一や撮り忘れが起きやすい。このスキルはマニュアル内の撮影指示を解析し、Playwrightで正確な状態を再現して自動撮影する。

## ユーザーから取得すべき情報

1. **マニュアルファイルのパス** — `📸 撮影指示` コメントが含まれたMarkdownファイル
2. **アプリURL** — ログインページ
3. **アカウント情報** — メールアドレス + パスワード
4. **撮影対象の範囲** — 全Chapter / 特定のChapterのみ
5. **ビューポートサイズ** — デフォルトは 1920x1080（フルHD、16:9）

## 撮影指示のフォーマット

マニュアル内の撮影指示は以下の形式で記述されている想定:

```markdown
![画像の説明](../../assets/{role}/images/filename.png)
<!-- 📸 撮影指示: 具体的な撮影内容の説明 -->
```

撮影指示から以下を読み取る:
- **保存先パス**: `![...](パス)` の括弧内
- **撮影内容**: `<!-- 📸 撮影指示: ... -->` の説明文
- **必要な画面状態**: URLパス、ページの状態（ログイン後、入力済み、スクロール位置など）

## 撮影プロセス

### ステップ1: 準備

1. マニュアルファイルを読み、全ての `📸 撮影指示` を抽出する
2. 撮影リスト（ファイル名 + 撮影内容）を一覧化し、ユーザーに確認を取る
3. Playwrightのビューポートサイズを統一する（デフォルト: 1920x1080）

### ステップ2: ログインと基本状態の準備

1. ログインページにアクセスする
2. アカウント情報でログインする
3. ログイン後のデフォルトページが表示されることを確認する

### ステップ3: 章単位での撮影

マニュアルの順番（Chapter順）に沿って撮影する。理由:
- 後のChapterの画面は、前のChapterの操作結果に依存する場合がある
- ログイン→ダッシュボード→各機能ページの順が自然な遷移になる

各スクリーンショットについて:

1. **ページ遷移**: 撮影指示に記載された画面に移動する
2. **状態再現**: 必要な操作を行う（フォーム入力、タブ切替、スクロールなど）
3. **待機**: ページの読み込み完了を待つ（2〜3秒）
4. **撮影範囲を判断**: 下記の基準に従い、ビューポート全体か要素切り抜きかを決める
5. **撮影**: Playwrightの `screenshot()` でファイルに保存する
6. **検証**: 撮影結果を確認し、意図した内容が写っているか確認する

### 撮影範囲の判断基準

撮影対象に応じて、ビューポート全体のスクリーンショットと要素単位の切り抜きを使い分ける。

**ビューポート全体で撮影する場合:**
- ページ全体のレイアウトを見せたいとき
- モーダルやダイアログが画面全体を覆っているとき（全画面モーダル等）
- 複数の要素の位置関係を示したいとき

**要素単位で切り抜く場合:**
- ボタン単体（「送信する」「新しい質問をする」等）を示したいとき
- 特定のセクションやカード1つだけを見せたいとき
- ページ全体を見せると対象の要素が小さすぎて伝わらないとき

### 要素切り抜き時の注意点

要素単位のスクリーンショットは、refの選び方で切り抜き範囲が大きく変わる。以下の基準で適切なrefを選ぶ。

**切り抜きが狭すぎる場合（内容が見切れる）:**
- 子要素ではなく親要素のrefを指定する。例えばフォーム入力欄だけでなく、フォーム全体のコンテナを指定する

**切り抜きが広すぎる場合（余白が大きい）:**
- タブパネル全体ではなく、その中の特定セクションのrefを指定する

撮影後は必ず画像を目視確認し、切り抜きが意図通りか検証する。問題がある場合はrefを変えて撮り直す。

### トースト通知の対処

操作後にトースト通知（成功メッセージ等）が画面上に表示されることがある。トーストが映り込んで画像が汚れる場合は、以下のJavaScriptで非表示にしてから撮影する。

```javascript
// トースト通知を非表示にする
await page.evaluate(() => {
  document.querySelectorAll('[role="status"], [data-sonner-toaster]')
    .forEach(el => el.style.display = 'none');
});
```

### ステップ4: 結果報告

撮影完了後、以下をユーザーに報告する:

| ファイル名 | 内容 | ステータス |
|----------|------|----------|
| `3-1-1-dashboard.png` | ダッシュボード全体 | ✅ 撮影済 |
| `3-2-1-basic-info.png` | 基本情報セクション | ✅ 撮影済 |
| `4-5-2-task-answer.png` | 課題の回答表示 | ⚠️ 要差替（ダイアログが被った） |

## マニュアル内容の検証と修正

撮影中にマニュアルの記述と実際のUI/操作が異なることに気づいた場合は、**マニュアルの記述を修正すること**。撮影指示に従って機械的に撮るのではなく、実際の画面と照合しながら進める。

例:
- 「画像をクリック」→ 実際は「画像を変更」ボタンをクリック → マニュアルを修正
- 撮影指示に書かれた要素が画面上に存在しない → マニュアルの手順を修正

## Playwrightで技術的に撮影不可能なケース

以下のようにPlaywrightでは撮影が技術的に不可能・困難なケースがある。これらに遭遇した場合は、**自己判断でプレースホルダー画像を作らず、ユーザーに報告し、代替案を提案すること**。

- **OSネイティブダイアログ**: ファイル選択ダイアログ（`<input type="file">`のクリックで開くOS標準ダイアログ）はPlaywrightでスクリーンショットを取れない
- **ドラッグ＆ドロップ中の状態**: 要素をドラッグ中のスクリーンショットは再現が困難
- **ホバー＋クリックの複合状態**: 特定の複合操作状態の再現が困難な場合がある

## 撮影のコツ

### ページの読み込み待ち
SPAでは画面遷移後にコンテンツが遅延ロードされることがある。遷移後は最低2秒待ってから撮影する。

### スクロール位置
ページ下部のコンテンツを撮影する場合、メインコンテンツ領域内でスクロールする必要がある場合がある（ページ全体のスクロールではなく）。

### ログイン状態の維持
Playwrightのセッション内ではCookieが維持されるため、一度ログインすれば連続撮影が可能。ただしセッションタイムアウトに注意し、長時間の撮影では適宜ログイン状態を確認する。

### ロール切替が必要な場合
複数ロールのスクリーンショットが必要な場合は、ロールごとにまとめて撮影する。ログアウト→再ログインで切り替える。

## 異常な画面状態の検知（最重要ルール）

撮影中に以下のような**正常系でない画面状態**を検知した場合は、**自己判断でプレースホルダー画像をコピーして進めず、必ずユーザーに確認を取ること**:
- ボタンが `disabled` で操作できない（例: 未採点の成果物がある場合の提出ボタン）
- 警告・エラーメッセージが表示されている（例: 担当コーチが対応していない）
- データが空で撮影指示の内容を再現できない（例: タスクが0件のカンバンボード）
- 期待する画面遷移が起きない（ダイアログが開かない等）

**理由**: プレースホルダー画像で進めると後戻りが発生し、大幅な時間のロスになる。異常に気づいた時点で即座にユーザーに報告し、対処方法（データ準備・別アカウント使用・スキップ等）を確認する。
