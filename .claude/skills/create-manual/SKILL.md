# Create Manual

Web アプリの機能情報から、ロール別の操作マニュアルを生成するスキル。

## 前提

- CLAUDE.md にアプリ情報、ロール一覧、出力先パスが記載されていること
- 探索結果（findings.md）が存在すること（CLAUDE.md の「出力先」に記載のパス）
- 上記がない場合は `/setup` → `/explore` の実行を案内して中断する

## プロセス

### ステップ 1: 入力の確認

1. CLAUDE.md から出力先パス（マニュアル、画像）を読み取る
2. 探索結果（findings.md）を読み込む
3. 対象ロールを確認する（ユーザーの指示、またはCLAUDE.md のロール一覧）

### ステップ 2: 構成案の提示

マニュアルの目次を先に作成し、ユーザーに提示する。承認を得てから本文の執筆に進む。

この段階でフィードバックを受けることで、書き直しのリスクを減らせる。

### ステップ 3: 本文の執筆

承認された構成に沿ってマニュアルを執筆する。

### ステップ 4: 保存

CLAUDE.md の「出力先」に記載されたパスに保存する。

## マニュアル構成ルール

### 文体

- **です/ます調**で統一
- **画面名・ボタン名**: 「」で囲む（例: 「ログイン」ボタン）
- **アクションベース**: 「〜するには」形式でセクションを構成
- **セクション番号は 1 始まり**（`## 1. はじめに` の形式）

### セクション粒度

1 つの操作フローに属する関連アクションは、同じセクションにまとめる。読者の視点で「この操作をするためにこのページを開く」という単位でセクションを切る。

**冗長な分割（避ける）:**
```
### 2-1. アイコンをクリックしてモーダルを開く
### 2-2. モーダルの画面構成を確認する
### 2-3. 全画面表示に切り替える
### 2-4. モーダルを閉じる
```

**適切な粒度:**
```
### 2-1. チャットボットを開く         ← 開く+全画面を1セクションに
### 2-2. 画面構成                     ← テーブルで要素一覧、閉じ方の補足もここ
```

判断基準:
- 同じ画面内の操作は同じセクションにまとめる
- 「開く→全画面にする→閉じる」のような表示制御は 1 セクションで十分
- 独立した操作フロー（「質問する」「過去質問を見る」等）は別セクションにする

### 画像の配置

手順の各ステップ直後に、そのステップに対応する画像 1 枚と撮影指示コメントを配置する。画像パスは CLAUDE.md の「出力先」に記載のパスを使う。

```markdown
### セクション名

1. ステップの説明文
![画像の説明](../../assets/{role}/images/filename.png)
<!-- 📸 撮影指示: 具体的な撮影内容 -->
2. 次のステップの説明文
![画像の説明](../../assets/{role}/images/filename.png)
<!-- 📸 撮影指示: 具体的な撮影内容 -->
```

**重要: ステップと画像の間に空行を入れない。** 空行を入れると番号付きリストが分断され、全ステップが「1.」にリセットされる。

**ステップ統合ルール:**
- アクション（クリック等）とその結果（画面表示）が同じ画面になる場合は、**1 つのステップにまとめて画像 1 枚**にする
- 例: 「サイドバーの『設定』をクリックします」+「設定画面が表示されます」→ 「サイドバーの『設定』をクリックすると、設定画面が表示されます。」+ 画像 1 枚
- 1 ステップ = 1 画像（重複なし）

**画像ファイル名**: `セクション番号-ステップ番号-簡潔な英語名.png`（例: `4-1-1-content-list.png`）

**📸 撮影指示の書き方**（`/capture-screenshots` がこのコメントを読み取って自動撮影する）:
- どの画面（URL/ページ名）か
- どの状態（初期表示/入力済み/クリック後など）か
- 何が見えるべきか
