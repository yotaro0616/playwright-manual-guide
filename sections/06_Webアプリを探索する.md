# Web アプリを探索する

## 🎯 このセクションで学ぶこと

- `/setup` でプロジェクトを初期設定する
- `/explore` で Web アプリを調査する
- 出力される調査レポート（findings.md）の読み方

---

## 🏃 Web アプリを探索する

### Step 1: プロジェクトを初期設定する

まず `/setup` を実行して、CLAUDE.md にあなたのアプリ情報を設定します。

```
/setup
```

Claude Code がアプリの URL やログイン情報、ロールなどを対話的に聞いてくれます。質問に答えていくだけで、CLAUDE.md とディレクトリ構造が自動で整います。

探索対象はどんな Web アプリでも OK です。社内ツール、学習サイト、自分が関わっているサービスなど、「操作マニュアルを作りたい」と思うものを選んでください。

> 💡 **試すアプリがない場合**: ログイン不要の公開サイト（例: お使いの会社の管理画面や、無料で試用できる Web サービス）でも試せます。ログインが必要な場合はテストアカウントを用意してください。

> ⚠️ **URL とログイン情報は必須です。** スキルはブラウザでアプリにログインして全画面を巡回するため、この情報がないと何もできません。

### Step 2: スキルを呼び出す

Claude Code で以下のように入力します。

```
/explore
```

自動的に以下の流れで探索が進みます。

1. CLAUDE.md から出力先パスを読み取り、探索結果ファイルを作成
2. アカウント情報でログイン
3. メニューやナビゲーションから全ページにアクセス
4. 各画面のボタン、フォーム、ポップアップなどを記録
5. 複数アカウントがあれば切り替えて繰り返し

> 💡 全体ではなく一部だけ探索したい場合は、追加指示を付けてください。
> ```
> /explore 設定画面まわりだけ探索して
> ```

### Step 3: 調査レポートを確認する

探索が完了すると、調査レポート（デフォルト: `exploration/findings.md`）が作成されます。レポートには以下の情報がまとまっています。

| セクション | 内容 |
|-----------|------|
| **メニュー構成** | 画面の一覧と URL |
| **機能詳細** | 各画面のボタンや入力欄と操作方法 |
| **画面間のつながり** | ある画面での操作が別の画面にどう反映されるか |
| **要確認事項** | AI が判断できなかった点（あなたへの質問） |

各項目には確信度マークが付いています。

| マーク | 意味 |
|--------|------|
| ✅ 確認済 | 実際に操作して確認した |
| ⚠️ 要確認 | 画面は見たが仕様に自信がない |
| ❓ 不明 | 操作できなかった |

**⚠️ と ❓ の項目は必ず目を通してください。** ここでの精度が次のマニュアル作成の品質に直結します。

---

## ✨ まとめ

- `/setup` でアプリ情報を CLAUDE.md に設定してから探索を開始する
- `/explore` で全画面を自動巡回し、調査レポートが出力される
- ⚠️・❓ の項目は確認して、必要なら情報を補足する
