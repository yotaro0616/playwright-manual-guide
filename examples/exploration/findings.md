## アクセス方法

1. ログイン後、サイドバーの「教材」をクリック

### 教材カードの構造 ✅ 確認済

各教材はカード形式で表示される。

| 要素 | 説明 |
|------|------|
| サムネイル画像 | 教材のカバー画像 |
| タイトル | 教材名（例: 「tutorial-1: プログラミング学習をはじめる前に🌱」） |

## 教材詳細ページの構造

### ページレイアウト ✅ 確認済

URL パターン: `/v2/user/{userId}/curriculums/{curriculumId}/chapters/{chapterId}/sections/{sectionId}/`

| 要素 | 位置 | 説明 |
|------|------|------|
| パンくずリスト | メイン上部 | 「教材一覧 > 教材」（教材一覧はリンク、教材は現在のページ） |
| ページタイトル | メイン上部 | 「教材」 |
| 教材サイドバー（左カラム） | メイン左 | 進捗率（%）+ Chapter 一覧 + 各 Chapter のセクション一覧 |
| 教材本文（右カラム） | メイン右 | Markdown 形式の教材コンテンツ |
| 読了ボタン | 本文下部 | 「この記事を読了する🔥」ボタン |

### 教材サイドバー（左カラム）✅ 確認済

| 要素 | 説明 |
|------|------|
| 進捗率 | 「0%」のように進捗をパーセントで表示 |
| Chapter 見出し | 例: 「chapter-1: プログラミング学習の心得とコツ」 |
| セクションリンク | Chapter 配下のセクション一覧。クリックでセクションに遷移。例: 「プログラミングとは」「エンジニアに必要な3つの力」 |

### 教材本文エリア ✅ 確認済

教材本文は Markdown 形式で、以下の要素が含まれる。

| 要素 | 説明 |
|------|------|
| セクション番号 + タイトル | 例: 「1-1-1: プログラミングとは」（h1） |
| 🎯 学習目標 | 「このセクションで学ぶこと」の箇条書き |
| 本文 | 見出し（h2, h3, h4）、段落、リスト、テーブル、引用ブロック、コードブロック等 |
| 💡 TIP | ヒントや補足情報 |
| ✨ まとめ | セクションの要点まとめ |

### 読了ボタンと読了後の動作 ✅ 確認済

| 要素 | 説明 |
|------|------|
| 「この記事を読了する🔥」ボタン | 本文の最下部に配置。未読了のセクションにのみ表示される |

**読了ボタンクリック後の動作:**

1. **読了完了モーダル** が表示される
   - メッセージ: 「お疲れ様でした！」「教材の学習が完了しました。素晴らしいですね！」
   - 「次の教材へ進む」ボタン — 次のセクションへ遷移
   - 「このページにとどまる」ボタン — モーダルを閉じて現在のページに留まる
   - 閉じるボタン（×）
2. **進捗率が更新** される（例: 6 セクション中 1 つ読了 → 0% から 17% に変化）
3. **左カラムのセクションリンクにチェックアイコン** が付く
4. **読了ボタンが消え**、代わりにナビゲーションが表示される
   - 「教材一覧へ戻る」ボタン
   - 「Next: （次のセクション名）」リンク

### 進捗率 ✅ 確認済

進捗率はセクション単位で計算される。各 Chapter 内のセクション数を分母として算出（例: 6 セクション中 1 読了 = 17%）。

## 操作フロー

### 教材を閲覧するフロー ✅ 確認済

1. サイドバーの「教材」をクリック → 教材一覧ページに遷移
2. 下にスクロールして「新教材」セクションを見つける
3. 教材カードをクリック → 教材詳細ページに遷移（最初の Chapter の最初のセクションが表示される）
4. 教材本文を読む
5. 左カラムのセクションリンクをクリックして別のセクションに移動
6. 「この記事を読了する🔥」ボタンをクリックして読了をマーク
7. 読了完了モーダルで「次の教材へ進む」または「このページにとどまる」を選択

## 共通 UI 要素

### AI 質問チャットボット ✅ 確認済

画面右下にフローティングウィジェットとして常時表示。タイトル入力・質問内容入力・「AIに質問する」ボタンで構成。過去の質問履歴も表示される。
