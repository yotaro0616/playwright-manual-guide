# Web アプリを探索する

## 🎯 このセクションで学ぶこと

- web-app-explorer スキルを呼び出して Web アプリを調査する
- 出力される調査レポート（findings.md）の読み方

---

## 🏃 Web アプリを探索する

### Step 1: CLAUDE.md にアプリ情報を記載する

スキルを呼び出す前に、プロジェクトの `CLAUDE.md` に探索したい Web アプリの情報を記載してください。これがないとスキルが動けません。

探索対象はどんな Web アプリでも OK です。社内ツール、学習サイト、自分が関わっているサービスなど、「操作マニュアルを作りたい」と思うものを選んでください。

> 💡 **試すアプリがない場合**: ログイン不要の公開サイト（例: お使いの会社の管理画面や、無料で試用できる Web サービス）でも試せます。ログインが必要な場合はテストアカウントを用意してください。

このリポジトリの `CLAUDE.md` にはテンプレートがあらかじめ用意されています。テキストエディタまたは Claude Code で開いて、**URL とアカウント情報をあなたのアプリのものに書き換えてください**（そのままでは動きません）。

```markdown
## アプリ情報

- URL: https://your-app.example.com/login
- アカウント:
  - メールアドレス: user@example.com
  - パスワード: password123

## 安全制約

- データの削除は行わない
- 設定の変更は行わない
```

ログインの種類が複数ある場合（例: 管理者と一般ユーザーなど）は、それぞれのアカウント情報を記載してください。

CLAUDE.md の編集は、Claude Code に頼むのが簡単です。

```
CLAUDE.md に以下を追記して

（上の内容をあなたのアプリ情報に書き換えて貼り付け）
```

もちろん、テキストエディタで直接 `CLAUDE.md` を開いて編集しても OK です。

> ⚠️ **URL とログイン情報は必須です。** スキルはブラウザでアプリにログインして全画面を巡回するため、この情報がないと何もできません。

### Step 2: スキルを呼び出す

Claude Code で以下のように入力します。

```
/web-app-explorer
```

自動的に以下の流れで探索が進みます。

1. `exploration/findings.md` を作成
2. アカウント情報でログイン
3. メニューやナビゲーションから全ページにアクセス
4. 各画面のボタン、フォーム、ポップアップなどを記録
5. 複数アカウントがあれば切り替えて繰り返し

> 💡 全体ではなく一部だけ探索したい場合は、追加指示を付けてください。
> ```
> /web-app-explorer 設定画面まわりだけ探索して
> ```

### Step 3: 調査レポートを確認する

探索が完了すると、`exploration/findings.md` が作成されます。レポートには以下の情報がまとまっています。

| セクション | 内容 |
|-----------|------|
| **メニュー構成** | 画面の一覧と URL |
| **機能詳細** | 各画面のボタンや入力欄と操作方法 |
| **画面間のつながり** | ある画面での操作が別の画面にどう反映されるか |
| **要確認事項** | AI が判断できなかった点（あなたへの質問） |

各項目には確信度マークが付いています。

| マーク | 意味 |
|--------|------|
| ✅ 確認済 | 実際に操作して確認した |
| ⚠️ 要確認 | 画面は見たが仕様に自信がない |
| ❓ 不明 | 操作できなかった |

**⚠️ と ❓ の項目は必ず目を通してください。** ここでの精度が次のマニュアル作成の品質に直結します。

---

## ✨ まとめ

- `CLAUDE.md` にアプリの URL・ログイン情報・安全制約を書いてから呼び出す
- `/web-app-explorer` で全画面を自動巡回し、`exploration/findings.md` にレポートが出る
- ⚠️・❓ の項目は確認して、必要なら情報を補足する
