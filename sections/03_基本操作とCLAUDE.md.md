# 基本操作と CLAUDE.md

## 🎯 このセクションで学ぶこと

- Claude Code にプロンプト（指示）を送る方法
- 権限の許可・拒否の判断
- CLAUDE.md でプロジェクトの文脈を伝える方法

---

## プロンプトの送り方

Claude Code を起動すると、ターミナルに入力欄が表示されます。ここに日本語でやりたいことを入力し、Enter を押すだけです。

```
このフォルダにある README.md の内容を教えて
```

Claude Code はあなたの指示を受け取ると、必要に応じてファイルを読んだり、コマンドを実行したりして回答します。

### 権限の許可

Claude Code がファイルの作成・編集やコマンドの実行を行う際、「この操作を実行してよいですか？」という確認が表示されることがあります。

- **Allow**（許可）: その操作を実行する
- **Deny**（拒否）: その操作を実行しない

内容を確認して問題なければ Allow を選んでください。

> 💡 毎回確認するのが面倒な場合は、「Always allow」を選ぶとそのセッション中は同じ種類の操作を自動で許可します。

## CLAUDE.md とは

**CLAUDE.md** は、Claude Code がセッション開始時に自動で読み込む**プロジェクトの説明書**です。

プロジェクトのフォルダ直下に `CLAUDE.md` というファイルを置いておくと、Claude Code は毎回このファイルを読んでから作業を始めます。つまり、「このプロジェクトでは何をしたいのか」「どんなルールがあるか」を毎回説明し直す必要がなくなります。

### CLAUDE.md に書くこと

| 項目 | 例 |
|------|-----|
| プロジェクトの目的 | 「○○の操作マニュアルを作成するプロジェクト」 |
| ルール・制約 | 「です・ます調で書く」「画像は assets/ に保存する」 |
| フォルダ構成 | どのフォルダに何があるか |
| アカウント情報 | アプリのログイン URL やテストアカウント |

### CLAUDE.md を作る

プロジェクトのフォルダで Claude Code を起動し、以下のように入力してみましょう。

```
/init
```

`/init` は、Claude Code がフォルダの中身を分析して CLAUDE.md のたたき台を自動生成するコマンドです。フォルダが空でも基本的な内容を作ってくれます。生成された内容を確認し、必要に応じて修正してください。

> 💡 ここでは CLAUDE.md の仕組みを知っておくだけで OK です。セクション 6 で実際にアプリの情報を書き込みます。

もちろん、テキストエディタで `CLAUDE.md` を手動で作成しても構いません。

> 💡 CLAUDE.md は Markdown 形式のテキストファイルです。見出し（`#`）や箇条書き（`-`）を使って書きます。

このリポジトリには、あらかじめ `CLAUDE.md` のテンプレートが用意されています。セクション 6 でアプリ情報を書き込むときに使います。

---

## ✨ まとめ

- Claude Code には日本語でやりたいことを入力するだけでよい
- 操作の実行時に権限の確認が出たら、内容を見て Allow / Deny を判断する
- **CLAUDE.md** をプロジェクトフォルダに置くと、毎回の説明が不要になる
- `/init` で CLAUDE.md のたたき台を自動生成できる
