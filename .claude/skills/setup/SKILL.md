# Setup

プロジェクトの初期設定を対話的に行い、CLAUDE.md とディレクトリ構造を整えるスキル。

## このスキルの役割

後続の `/explore`、`/create-manual`、`/capture-screenshots` はすべて CLAUDE.md の情報を参照して動作する。このスキルで CLAUDE.md を正しく設定しておくことで、3 つのスキルがスムーズに連携する。

## プロセス

### ステップ 1: ユーザーにヒアリングする

以下の情報を対話的に確認する。不明な項目はデフォルト値を提案する。

| 項目 | 必須 | デフォルト |
|------|------|-----------|
| アプリ URL（ログインページ） | はい | — |
| アカウント情報（メール + パスワード） | はい | — |
| ロール一覧（管理者、利用者 等） | はい | — |
| 安全制約（やってはいけない操作） | いいえ | 「データの削除・設定の変更は行わない」 |
| 探索結果の出力先 | いいえ | `exploration/findings.md` |
| マニュアルの出力先 | いいえ | `docs/{role}/manual.md` |
| 画像の保存先 | いいえ | `assets/{role}/images/` |

### ステップ 2: CLAUDE.md を更新する

ヒアリング結果をもとに CLAUDE.md を更新する。既存の CLAUDE.md がある場合はその内容を保持しつつ、以下のセクションを追加・更新する。

```markdown
## アプリ情報

- URL: （ヒアリング結果）
- アカウント:
  - ロール名:
    - メールアドレス: xxx
    - パスワード: xxx

## ロール

- ロール 1: 説明
- ロール 2: 説明

## 安全制約

- （ヒアリング結果）

## 出力先

| 成果物 | パス |
|--------|------|
| 探索結果 | exploration/findings.md |
| マニュアル | docs/{role}/manual.md |
| 画像 | assets/{role}/images/ |

## ワークフロー

1. `/explore` — アプリを探索して調査レポートを作成
2. `/create-manual` — 調査結果からマニュアルを生成
3. `/capture-screenshots` — マニュアルの📸指示からスクリーンショットを自動撮影
```

### ステップ 3: ディレクトリ構造を作成する

CLAUDE.md の「出力先」に基づいてディレクトリを作成する。

### ステップ 4: 完了報告

設定内容のサマリーを表示し、次のステップ（`/explore` の実行）を案内する。
